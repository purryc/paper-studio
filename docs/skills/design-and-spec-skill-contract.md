# Design And Spec Skill Contract

## frontend-design

Used to shape the current Paper Studio UI:

- one-screen creative console
- liquid-glass visual language
- camera chips instead of nested source controls
- compact capture/review/intent/generate panels
- inline generated results

It is not a runtime dependency.

## open-spec

Used as the on-demand durable planning workflow for this prototype.

This repo does not require Spec Kit by default. Small fixes can proceed directly, but durable decisions should be reflected in README, docs, specs, or agent notes.

It is not a runtime dependency.

## deliverable-director

Used as design precedent for editable flowchart delivery:

- semantic Mermaid source of truth
- backend-controlled orthogonal rendering
- 90-degree Manhattan connectors
- native editable PowerPoint shapes
- minimum readable font sizes

It is not a runtime dependency.
